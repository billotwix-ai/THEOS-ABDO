📌 ما هو هذا المشروع؟

Theos Compiler Bot هو بوت تيليقرام احترافي يسمح للمطورين برفع مشاريع iOS Tweak المبنية على Theos، وكمبايلها تلقائياً على السيرفر، وإرجاع الملف الجاهز (deb / dylib / framework) مباشرة في المحادثة.

لا يحتاج المستخدم أي إعداد على جهازه — فقط يرفع الملف المضغوط ويستلم النتيجة خلال دقائق.

📌 What is this project?

Theos Compiler Bot is a professional Telegram bot that lets iOS tweak developers upload their Theos projects as compressed archives, automatically compile them on a remote server, and receive the output file (deb / dylib / framework) directly in chat.

No local setup required for end users — they simply upload a zip and receive the compiled result within minutes.

⚡ Features
📦
Multiple archive formats zip, rar, 7z, tar.gz, tar.bz2
⚙️
Parallel builds Up to 3 concurrent compilations
🔄
Auto-path patching Fixes THEOS path in Makefile
❌
Build cancellation Cancel compilation at any time
📋
Full error logs Sends build_error.log on failure
🛡️
Rate limiting 1 minute cooldown per user
🔒
Private logging All projects logged to private group
🔁
Auto-restart Runs via systemd, always on
🖥️ VPS Requirements

The bot requires a Linux VPS. Here are the specs:

Resource Minimum MIN Recommended REC Reason
🖥️ CPU 2 Cores 4 Cores Compilation is CPU-heavy, especially with 3 simultaneous builds
💾 RAM 2 GB 4 GB clang needs sufficient memory; each build takes ~500MB
💿 Storage 20 GB SSD 40 GB SSD Theos + SDKs + toolchains ≈ 10GB, plus space for builds
🌐 Network 100 Mbps 500 Mbps+ Fast upload/download of user project files
🐧 OS Ubuntu 20.04 LTS or Ubuntu 22.04 LTS Best compatibility with Theos
🏠 Location Any (US / EU / Asia) Choose a server close to your main user base
⚠️ Do not use a 1GB RAM VPS — compilation will fail due to insufficient memory.
🚀 Installation Steps on VPS
  • 1
    Prepare the server
    Ubuntu 22.04 on VPS, SSH in and update packages
    # Update packages sudo apt-get update && sudo apt-get upgrade -y sudo apt-get install -y git curl make clang python3 python3-pip unrar p7zip-full
  • 2
    Install Theos from theos.dev
    Official installer fetches Theos + toolchains + SDKs automatically
    export THEOS=/home/smartpepole/theos bash -c "$(curl -fsSL https://raw.githubusercontent.com/theos/theos/master/bin/install-theos)"
  • 3
    Add to PATH
    So the server recognizes Theos in every session
    echo 'export THEOS=/home/smartpepole/theos' >> ~/.bashrc echo 'export PATH=$THEOS/bin:$PATH' >> ~/.bashrc source ~/.bashrc
  • 4
    Configure token in bot.py
    Get a token from @BotFather and edit the file
    TELEGRAM_TOKEN = "YOUR_TOKEN_HERE" LOG_GROUP_ID = -100XXXXXXXXXX # Your private group ID
  • 5
    Run the install script
    Installs Python deps and registers bot as a systemd service
    chmod +x install.sh && sudo ./install.sh
  • 6
    Verify it's running
    Check the service status and live logs
    sudo systemctl status theos-bot sudo journalctl -u theos-bot -f
🔄 How does the bot work?
📤 User uploads a compressed project archive (zip/rar/7z)
↓
📥 Bot downloads the file to the VPS
↓
📦 Extracts archive and locates the Makefile
↓
🔧 Auto-patches THEOS path in Makefile
↓
⚙️ Runs make package and monitors output
↓
✅ Sends .deb / .dylib / .framework back to the user
📁 Project Structure
theos-bot/ ├── bot.py # Main bot (full retry logic) ├── bot66666666.py # Simpler alternate version ├── requirements.txt # python-telegram-bot, patool, pyunpack ├── install.sh # Auto install & start script ├── theos-bot.service # systemd unit for persistent service └── README.md # Full documentation
🛠️ Management Commands
# Restart sudo systemctl restart theos-bot # Stop sudo systemctl stop theos-bot # Live logs sudo journalctl -u theos-bot -f # Enable auto-start on reboot sudo systemctl enable theos-bot
